Deploying to GitHub Pages - Step by Step

1

Create a GitHub Account

If you don't have one, go to github.com and sign up for a free account.

2

Create a New Repository

Click the "+" icon in the top right and select "New repository".

Repository name: fema-p154-screening
Description: FEMA P-154 Rapid Visual Screening Tool
Public: ✓ (required for free GitHub Pages)
Initialize with a README: ✓
3

Upload the HTML File

In your new repository, click "Add file" → "Upload files".

Drag and drop your HTML file (named index.html for the main page).

# Alternatively, use Git commands:
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/your-username/your-repo-name.git
git push -u origin main
4

Enable GitHub Pages

Go to Settings → Pages → Source → select "main" branch → Save.

Your site will be published at: https://your-username.github.io/your-repo-name

5

Wait for Deployment

It may take a few minutes for your site to be available. Refresh the page after a few minutes.

6

Share the URL

Once deployed, share the GitHub Pages URL with your inspectors.

Example: https://your-username.github.io/fema-p154-screening
